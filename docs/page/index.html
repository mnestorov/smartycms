Introduction Last updated: 2020-09-12

SmartyCMS is a Laravel admin panel CRUD (create, read, update and delete) package that can help you get your administration panel in minutes.

About SmartyCMS

SmartyCMS will provide you with a visual interface for the admin panel (the HTML, the CSS, the JS). It pulls in the excellent UI theme, with our own design adds authentication functionality & bubble notifications. When you decide to build a custom feature for your admin panel, you already have the HTML blocks for the UI, and it will look good

Modules

At this moment we support these modules:
Pages (pages, page elements, galleries, subpages)
Blog (posts, categories, comments)
Shop (products, categories, comments, orders, stock)
Galleries ---
Images and Files ---
Locations and Maps (places)
Leads (contacts, subscriptions)
Multiple admins ---

Once you have your administration panel up, you can easily put all of those elements wherever you want in your application files.

Installation

Detailed instructions on how to install SmartyCMS super fast.

Install using Composer

$ composer require smartystudio/smartycms

In Laravel 5.5+, with Package Auto Discovery it should all be set automatically. For < 5.5, follow these instructions after composer finishes package installation:

Add the service provider to the 'providers' array in config/app.php for Laravel 5.4 and lower.

SmartyStudio\SmartyCms\AdminServiceProvider::class,

If you want to use this package as a facade, add this line to the $aliases array in config/app.php.

'Cms' => SmartyStudio\SmartyCms\Facades\Cms::class,

Start package installation by running install command below:

$ php artisan smartycms:install

If you want to install package again from scratch, just delete the config/smartycms.php file and drop database, then run install command again.

If our package update throws composer error, try updating dependencies manually with commend below:

$ php artisan smartycms:update

Note

This installation uses migrations, so you must run it from machine that has access to your database.


For instance, if you use Vagrant, you will have to do vagrant ssh first, go to your project directory, and run this install command. The same way you run your standard Laravel's migration command.

Publish Assets

Now, when your package's users execute the vendor:publish command, your assets will be copied to the specified publish location. Since you will typically need to overwrite the assets every time the package is updated, you may use the --force flag:

$ php artisan vendor:publish --tag=public --force

Compiling Assets (Mix)

It's important if you make some changes on style files, to compile assets after you're done with all style changes.

Install npm or yarn

npm

$ cd src/
$ npm install

yarn

$ cd src/
$ yarn install

Build

npm build

$ cd src/
$ npm run production

yarn build

$ cd src/
$ yarn prod

Page

Instance

You could access SmartyCMS Page model instance with Cms::page().

Cms::page();           /* this will return you appropriate Query Builder */
Cms::page($page);      /* this will return the Page Model, where `$page` can be page ID or page slug. */
Cms::page()->menu();   /* generate nested menu from `Pages` */

This will generate your HTML.

You can style your menu like this:

.nav-list { 
   li > ul { 
      padding-left: 20px; 
      padding-right: 20px; 
   } 
}
Cms::page()->tree();  /* generates menu array from `Pages` */

Elements

Cms::page($page)->elements();    /* returns you all page elements as relationship `hasMany`. */
Cms::element($element_key);      /* returns PageElement Model */
echo Cms::element($element_key); /* returns content of the element, like Cms::element($element_key)->content */

SEO Meta

Implementation

getMeta() returns array of meta keys and it can be sent directly to blade template. Use $title, $description and $keywords variables in your blade layouts.

return view('about', CMS::page('about')->getMeta());

Blog

Model Instance

You could access SmartyCMS Blog model instance with Cms::blog().

Cms::blog()->posts      /* return you an appropriate Query Builder */
Cms::blog()->categories /* same as above */
Cms::blog()->comments   /* same as above */

You could call a method to get Query Builder, or you could call a property to get a Collection.

Posts

Cms::blog()->posts you get a Query Builder so you have a basic idea how it works.

Let's assume you want to retrieve all posts from database in your front view and you have a BlogController.php file in your app/Http/Controllers folder and the first method is index().

app/Http/Controllers/BlogController.php

public function index()
{
    $posts = Cms::blog()->posts;
    return view('blog.index', compact('posts'));
}

Let's assume you want to retrieve single post from database in your front view and you have a BlogController.php file in your app/Http/Controllers folder and there is a show() method.

app/Http/Controllers/BlogController.php

public function show($id)
{
    $post = Cms::blog()->posts()->find($id);
    return view('blog.show', compact('post'));
}

In case you want to create a Post outside SmartyCMS admin panel, these are all available columns.

Cms::blog()->posts()->create([
    'blog_category_id', (int default(null))
    'uri_id', (string)
    'title', (string)
    'thumb', (string)
    'content', (string)
    'excerpt', (string)
    'visible', (int default(0))
    'meta_title', (string)
    'meta_description', (text)
    'meta_description', (text)
    'meta_keywords', (string)
]);

If you want to get all comments for single post.

Cms::blog()->posts()->find($post_id)->comments   /* return you a relationship hasMany */
Cms::blog()->posts('my-first-blog')->comments    /* same as above */

If you want to get Category for single post.

Cms::blog()->posts()->find($post_id)->categories    /* return a relationship belongsTo */
Cms::blog()->posts('my-first-blog')->categories    /* same as above */

And this is how to retrieve post data in your front view.

resources/views/blog/index.blade.php

@foreach($posts as $post)
    {{ $post->title }}
@endforeach

Categories

Cms::blog()->categories /* you get a Query Builder */
Cms::blog()->posts      /* same as above */

Comments

Cms::blog()->comments /* you get a Query Builder */
Cms::blog()->posts    /* same as above */

Get post for single comment:

Cms::blog()->comments()->find($comment_id)->post /* return you a relationship belongsTo */

Write comment to post:

Cms::blog()->posts('my-first-blog')->comments()->create([
    'name', (string)
    'email', (string)
    'content', (text)
    'approved', (int) (default 0)
});

Shop

Instance

You can access SmartyCMS Shop instance with Cms::shop().

Cms::shop()->products() or Cms::shop()->categories() or Cms::shop()->comments() or Cms::shop()->orders() we will return you appropriate Query Builder.

You can call a method to get Query Builder, or you can call a property to get a Collection.

Products

Cms::shop()->products() will return you Query Builder, without brackets products will return Collection.

For single product Cms::shop()->products()->find($product_id) we are providing you with a few methods.

Every product belongsTo single category on category() method.

Every product has Gallery which belongsTo on method gallery().

Every product has multiple comments on relationship hasMany on comments() method.

Every product has multiple similar products on similar() method.

Product categories

Cms::shop()->categories() will return you Query Builder, without brackets categories will return Collection.

Single product categories have hasMany relationships on method products().

Cms::shop()->categories()->find($category_id)->products()

Product comments

Cms::shop()->comments() will return you Query Builder, without brackets comments will return Collection.

Single comments have belongsTo relationship on method product().

Cms::shop()->comments()->find($category_id)->product()

Add comment on product:

Cms::shop()->comments()->create([
    'product_id', (int)
    'name', (string)
    'email', (string)
    'message', (text)
    'approved', (int) (default 0)
]);

Orders

Cms::shop()->orders() will return you Query Builder, without brackets orders will return Collection.

    'shipment_price' => 100,

Example:

/* Add item to order */
Cms::shop()->orders()->find($order_id)->items()->create([

status() - return order status

items() - return order items

Galleries

Instances

With SmartyCMS you could easily manipulate with Galleries.

To reach all gallery types:

Cms::gallery()->get()

Get single gallery:

Cms::gallery($gallery_key)

Available methods:

images() - return collection of images

url() - all our images and files are stored in storage/app path so we need to generate URL from storage_key.

Only tested with the local file system (Storage) so feel free to send a review for s3 or any other driver.

Example:

foreach(Cms::gallery('foo')->images as $image) {
    echo '<img src="$image->url">';
}

Images and Files

Instances

With SmartyCMS you could manipulate with images and files.

All our files are stored at storage/app and saved in our database by storage_key so we need to provide you a URL to file. It is tested for Local storage, but feel free to send a review on another storage driver.

To generate link to a file use:

Cms::getFile($file_name)

To generate an progressive image with specific dimensions use:

Cms::getImage($image, $width, $height)

You can use only width or height to resize the image to given dimension, keeping the aspect ratio of the image. If both are given, image will be fitted in that canvas, also preserving the aspect ratio.


Note

Some of our modules, like Gallery, have built in url() method that can return a URL of the image or a file. This is useful because we store storage key as a source.


Locations and Maps

Instances

An easy way to manipulate with Locations and Maps with our Cms facade.

Please set your Google API key in our config/smartycms.php in array google_map_api for our map

Possibility to add a custom marker, thumb image or image for every location.

Locations

Get all locations:

Cms::locations()

Get single location:

Cms::location($location_key)

Map

Get all maps:

Cms::maps()

Get single map:

Cms::map($map_key)

Get locations for map:

Cms::map($map_key)->locations /* return hasMany relationship */

Leads

You could access SmartyCMS package Lead instance with Cms::lead().

You could save multiple fields at once and manipulate with them. This opportunity is great for a contact page and we will serialize all your data.

Everything that we expect of you is an instance of Illuminate\Http\Request

i.e. $request->all().

If you provide us with email and full_name in the request we will send a welcome message to the user.

To set up custom message visit leads/settings.

To start using leads, place code, as you could see below, as post method in your controller:

Cms::lead()->subscribe($request->all())

Don't forget to use Cms in your controller.

In case that you want to unsubscribe specific user, use:

Cms::lead()->unsubscribe($colunm, $value)

i.e. Cms::lead()->unsubscribe('email', 'johndoe@example.com').
